# Hard Rules

How work gets done in this system. Check output against these before delivering.

## Truth

1. **Never invent facts.** No fabricated numbers, citations, quotes, names, or results. If something is unknown, write a labeled placeholder like [FILL IN: the missing fact] and ask.
2. **Cite sources.** Skill files carry source and date at the top. Claims in notes say where they came from. School or work citations follow the required format, checked against real sources.
3. **Confidentiality.** [FILL IN: what stays private for you. Examples: employer internal material, names of people who confided in you, family matters. State the rule and how to genericize, e.g. "a mid-size firm in my city" instead of the name. If nothing applies yet, write "none yet" and add to this when it changes.]

## Time

4. **Date everything.** Brain notes are named `YYYY-MM-DD_short-slug.md`. Facts change; dates tell you which version is current.
5. **One fact per note.** Small, dated, linkable notes beat long documents that go stale as a unit.

## Structure

6. **Wikilinks connect notes.** Link text matches the target filename exactly so links resolve in Obsidian. Link where following it teaches something real, usually 2 to 5 links per note.
7. **INDEX.md stays current.** Every new brain note gets one line in the index. Renaming or deleting a note includes fixing its index line and every inbound wikilink in the same edit, so no broken links are left behind.
8. **Quality gates run before delivery.** Every directive ends with a checklist. Work that fails a gate gets fixed before the owner sees it.
