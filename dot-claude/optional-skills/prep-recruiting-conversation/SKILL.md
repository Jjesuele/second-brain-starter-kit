# Prep Recruiting Conversation

## Overview

The workflow lives in the vault at `directives/optional/recruiting_prep.md` (Gather → Angle → Produce → Save). Read and follow it. This skill governs the two judgments that decide whether the prep is worth anything: **every firm fact must be real and sourced, and every talking point must be something only this user could say.** A fabricated deal name in an interview or a generic ambition line in an email are the two failures that actually cost offers.

## Procedure

1. **Check the brain first.** Search the vault's `brain/` for prior notes on the target (person, firm, or related firms) before researching from scratch.
2. **Gather with receipts.** Every firm fact (deals, size, strategy, names, offices) comes from a checkable source: firm site, press release, credible coverage. Write the source next to each fact in the working notes. If public info is thin, say so plainly and prep general questions instead of padding (this is the directive's own edge case).
3. **Angle through their real story.** Pick 2-3 talking points connecting the target to something the user actually did, drawn from `context/me.md` and concrete stories in `brain/references/story-bank.md`. **The test for each talking point: could any other candidate say this sentence? If yes, it fails.** "I built the rent-roll tab for a tear-down" passes. "I'm passionate about the industry" fails.
4. **Produce by conversation type,** per the directive: firm research → one-page brief with "why this firm"; coffee chat → 5 questions + 30-second intro; interview → likely questions with answer outlines + technical topics; application → talking points. Questions must not be answerable from the firm's homepage. Run edit-into-voice on anything the user will send or say aloud.
5. **Save per the directive:** dated `brain/notes/YYYY-MM-DD_recruit-<target>.md`, wikilinks, INDEX.md line. After the real conversation happens, append what was said and follow-ups to the same note. If the session surfaced a new concrete story, add it to the story bank.

## Confidentiality

Nothing employer-confidential goes into external-facing content. Work experience gets genericized per the vault's `context/standards.md`: the type of work and market, never internal names or figures.

## Judgment Rubric

- [ ] Zero invented firm facts; every deal, number, and name has a written source
- [ ] Each talking point contains a detail only this user could say
- [ ] Questions are not answerable from the firm's homepage
- [ ] External content passes the edit-into-voice rubric
- [ ] Brain note saved, indexed, linked

## Pushback Rules

- The story-bank slot needed is still a placeholder → **stop and ask the user for the real story. Never write a plausible one.** A story they cannot back up in a live conversation is a liability, not prep.
- Asked for volume outreach (many near-identical emails) → push back once: the directive's design is tailored contact, and recruiters read generic blasts as generic. If they still want volume, tailor at minimum the first line and one talking point per recipient.
- A firm fact cannot be verified → present it as "reportedly" with the source, or leave it out. Never state it flat.

## Self-Check

Run the directive's QA gate. Then confirm: every firm fact has its source beside it, external text passed the edit-into-voice checks, brain note and INDEX line both exist.
